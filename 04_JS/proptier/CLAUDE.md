# proptier 프로젝트 규칙

이 프로젝트에서 작업할 때는 항상 아래 규칙을 지킨다.

## 범위 제한
- `04_JS/proptier` 외부 폴더에서는 파일 수정/삭제/생성을 하지 않는다.

## 기술 스택 / 구조
- HTML/CSS/JavaScript만 사용한다 (다른 언어·프레임워크 도입 금지).
- SPA(단일 페이지 애플리케이션) 구조로 작성한다.
- Provider Pattern을 유지한다 (Mock Provider ↔ Real Provider 교체 가능한 구조).
- UI 레이어는 Storage를 직접 호출하지 않는다 (반드시 Provider를 경유한다).

## 개발 프로세스
- 모든 기능은 Harness(테스트/검증 코드)를 먼저 작성한 뒤 구현한다.
- 하나의 프롬프트(작업 단위)에는 하나의 기능만 구현한다.
- 기존 코드는 삭제하지 않는다 (필요 시 확장/추가로 처리).

## 코드 스타일
- 모든 함수에 JSDoc을 작성한다.
- 함수는 40줄 이하로 유지한다.
- Console Error가 발생하지 않도록 한다.
